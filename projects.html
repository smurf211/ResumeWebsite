My Role

Project Difficulties & Solutions

Getting the logic to work correctly in the code was not so difficult. Making the math I was calculating based on the size of each block, and then translating that into the code which would display it in the JavaFX GUI was the difficult part. CPU Scheduler

CPU Scheduler

FCFS/SJF/Priority


View Source

Tech Used

  • ◦ Java
  • ◦ JavaFX
  • ◦ Eclipse
  • ◦ Scenebuilder
  • ◦ OS Scheduler

My Role

Project with pre-defined specifications and requirements. Small scope so all aspects were handled myself. Additional features were added as I saw necessary.

Project Difficulties & Solutions

This application allowed for up to 10 processes to be entered by the user to select one of three pre-defined CPU scheduler algorithms. First Come First Serve, Shortest Job First, and Priority Scheduling. Creating the algorithm to calculate the order of the processes based on SJF was the most difficult. This was because I had to consider the burst time, arrival time and how that effected the wait times of subsequent processes. The application features dynamic switching between algorithms and number of processes and allows for custome user input for all variables.

Features

  • FCFS/FJS/Priority scheduler simulation
  • variable burst, arrival and wait times
  • Hashing
  • Multithreading
  • Responsive GUI
